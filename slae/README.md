# Решение систем линейных алгебраических уравнений (СЛАУ)

Проект реализует лабораторную работу №1: разработку численного решателя СЛАУ с несколькими методами, консольным интерфейсом и графическим приложением на Qt.

## Возможности

- Класс `LinearSystemSolver`, принимающий матрицу коэффициентов `A` и вектор правых частей `b`.
- Поддержка методов из технического задания:
  - LU‑разложение.
  - Метод простых итераций.
  - Метод Якоби.
  - Метод Гаусса–Зейделя.
  - Метод наискорейшего спуска (для SPD‑матриц).
  - Метод сопряжённых градиентов (для SPD‑матриц).
- Консольный сценарий (`main.py`) с пошаговым вводом системы.
- Qt‑интерфейс (`gui.py`), запускаемый через оболочку `run_gui.sh`.
- Набор автоматических тестов с 100% покрытием для `solver.py`.

## Структура

```
main.py        # консольный запуск решателя
gui.py         # Qt‑приложение
run_gui.sh     # оболочка для запуска GUI (автоматически готовит .venv)
solver.py      # реализация класса LinearSystemSolver и всех методов
methods.py     # описания доступных методов для UI
tests/         # тесты (unittest)
requirements.txt
```

## Требования

- Python 3.10+.
- Виртуальное окружение с пакетами из `requirements.txt` (numpy, PyQt5).

## Быстрый старт

### Консоль

```bash
python3 main.py
```

1. Введите размер системы `n`.
2. Заполните матрицу `A` построчно.
3. Укажите вектор `b`.
4. Выберите метод решения. При выборе итерационных методов задайте дополнительные параметры.

### Графический интерфейс (Qt)

```bash
./run_gui.sh
```

- При первом запуске скрипт создаст `.venv` и установит зависимости.
- Затем откроется окно, где можно:
  1. Задать размер системы и сгенерировать таблицы.
  2. Заполнить матрицу `A` и вектор `b`.
  3. Выбрать метод — необходимые параметры появятся автоматически.
  4. Запустить решение и увидеть результат в нижней панели.
- Прерывание `Ctrl+C` в терминале корректно завершит GUI.

## Тесты и покрытие

```bash
python3 -m unittest discover
python3 -m coverage run -m unittest discover
python3 -m coverage report -m solver.py
```

`solver.py` покрыт тестами на 100% (включая проверки ошибок и сходимости).

### В GitHub Actions

Workflow `.github/workflows/ci.yml` запускается при push и pull request. Он выполняет тесты, строит HTML-отчёт покрытия и прикрепляет его как артефакт.

Как посмотреть отчёт на GitHub:

1. Откройте вкладку **Actions** репозитория и выберите последний запуск workflow `CI`.
2. В блоке **Artifacts** скачайте архив `coverage-html`.
3. Распакуйте архив локально и откройте файл `htmlcov/index.html` в браузере.
